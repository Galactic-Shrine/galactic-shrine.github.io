About Galactic-Shrine Standards

À propos de Galactic-Shrine Standards

Galactic-Shrine Standards est le portail de référence pour les standards internes de documentation et de codage de l’écosystème Galactic-Shrine. Il centralise les conventions, la structure documentaire, les règles de nommage et les décisions transversales destinées à garder une base cohérente dans le temps.

Portée

Un socle commun

Le portail couvre un noyau transversal Galactic-Shrine puis des standards spécifiques par technologie : C#, C++, PHP, JavaScript, SCSS, HTML et Twig.

Maintenabilité

Des règles stables

Chaque standard est pensé pour aider la review, réduire les divergences de style et rendre les projets plus lisibles, même quand ils grossissent.

Écosystème

Une logique Galactic-Shrine

Le portail reflète tes conventions réelles : structure harmonisée, pages HTML lisibles, packs ZIP, palette du site et règles de codage assumées.

Mission

Pourquoi ce portail existe

  • Centraliser la documentation normative dans un seul point d’entrée public et propre.
  • Diffuser les standards sans dépendre d’un environnement serveur dynamique.
  • Rendre lisible la progression de tes conventions via un changelog consultable.
  • Préparer une base exploitable à la fois pour les projets open source, internes et futurs produits Galactic-Shrine.

Principes

Les 4 principes directeurs

Le portail et les standards suivent une ligne éditoriale claire, utile à long terme.

1. Cohérence avant préférence individuelle

Le but n’est pas de refléter les habitudes par défaut d’un langage, mais de garder une base homogène dans l’ensemble Galactic-Shrine.

2. Lisibilité avant astuce

Le code, la documentation et la navigation du portail privilégient les structures simples, explicites et faciles à relire.

3. Documentation exploitable

Les standards doivent être consultables rapidement, en HTML comme en Markdown, avec une recherche qui reste fonctionnelle sans backend.

4. Industrialisation progressive

Le portail reste simple à déployer aujourd’hui, tout en préparant une montée en gamme vers un domaine dédié, des pages additionnelles et une meilleure gouvernance documentaire.

Cycle de publication

Comment les standards évoluent

Étape 01

Rédaction

Règles, exemples, portée et exceptions sont formalisés dans les fichiers sources Markdown.

Étape 02

Harmonisation

Les sections, noms, formulations et structures sont alignés entre les standards.

Étape 03

Publication

Le portail HTML, les packs ZIP, le changelog et les fichiers de déploiement sont mis à jour ensemble.
