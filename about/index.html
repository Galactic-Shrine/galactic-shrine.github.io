À propos de Galactic-Shrine Standards

Un portail de référence pour les standards internes de documentation et de codage de l’écosystème Galactic-Shrine.

Un socle commun

Le portail couvre un noyau transversal Galactic-Shrine puis des standards spécifiques par technologie : C#, C++, PHP, JavaScript, SCSS, HTML et Twig.

Des règles stables

Chaque standard aide la review, réduit les divergences de style et garde les projets lisibles même lorsqu’ils grossissent.

Une logique Galactic-Shrine

Structure harmonisée, pages HTML lisibles, packs ZIP, palette du site et règles de codage assumées.

Pourquoi ce portail existe

  • Centraliser la documentation normative dans un seul point d’entrée public et propre.
  • Diffuser les standards sans dépendre d’un environnement serveur dynamique.
  • Rendre lisible la progression de tes conventions via un changelog consultable.
  • Préparer une base exploitable pour les projets open source, internes et futurs produits Galactic-Shrine.

Principes directeurs

Les 4 principes directeurs

Le portail et les standards suivent une ligne éditoriale claire, utile à long terme.

1. Cohérence avant préférence individuelle

Le but n’est pas de refléter les habitudes par défaut d’un langage, mais de garder une base homogène dans l’ensemble Galactic-Shrine.

2. Lisibilité avant astuce

Le code, la documentation et la navigation du portail privilégient les structures simples, explicites et faciles à relire.

3. Documentation exploitable

Les standards restent consultables rapidement, en HTML comme en Markdown, avec une recherche qui fonctionne sans backend.

4. Industrialisation progressive

Le portail reste simple à déployer aujourd’hui tout en préparant une montée en gamme vers un domaine dédié et une meilleure gouvernance documentaire.
